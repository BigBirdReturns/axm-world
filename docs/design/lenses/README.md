# RoDo(h) orbital design passes

These passes audit the same playable-world slice from distinct decision systems. They are not visual-reference packs and contain no third-party assets.

- [Sprite Corpus Lens pass](./sprite-corpus-orbital-pass-v0.1.md) — native-scale embodiment, state, motion, terrain, and cross-asset ecology.
- [Aspirational Power Lens pass](./aspirational-power-orbital-pass-v0.1.md) — capability truth, actors, proof architecture, custody, and product communication.

Both passes select mechanisms rather than source aesthetics. Their shared next implementation target is **Embodiment Contract v1**: one cartridge/theme-owned contract that binds 2D dolls, the 3D traveler, motion state, place transformation, and custody proof without moving engine truth into presentation code.

## Evidence vocabulary

- `Observed` identifies behavior or source code present in this repository.
- `Inference` identifies a diagnosis derived from more than one observed seam.
- `Proposal` identifies new work. It is not a claim about source creators or reference organizations.

## Release boundary

No source sprite binary, campaign asset, layout, code, trade dress, character, pose, palette, or composition is included. Platform treatment remains interpretive. “Nintendo + Roblox” is an internal product constraint—polished legibility plus cartridge extensibility—not a request for visual imitation.
