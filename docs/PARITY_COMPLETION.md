# Bundled product parity: inventory, sequence, and completion contract

## Current scope

Parity means every bundled local-first cartridge receives the guarantees its authored grammar requires through one Arc engine and one holder-owned run contract.

1. Canonical Arc law and deterministic full-campaign reachability.
2. Cold entry, authored opening, first action, consequence, ledger, reload, run export, fresh-context import, and exact resume.
3. Board, Map, Hall, Encounter, Aperture, and Globe on desktop and mobile, plus a registered grammar-specific representation where the cartridge declares one.
4. Cartridge-owned roles, people, motifs, material treatment, and procedural palette.
5. One runtime-loaded hero environment, one lead portrait, one framing or depth asset, and original-authorship provenance for each first-party reference at the production dignity floor.
6. EN and zh-Hant runtime chrome, visible keyboard focus, reduced motion, local sound control, forced-colors and high-contrast behavior, and mobile target floors.
7. Neutral white-label fallback for imported cartridges, with no borrowed identity or receiver-authored law.

## Completion decision

The local-first parity gate passes only when Arc and World normal suites, builds, exact vendoring, full-campaign custody tests, production-asset contracts, and desktop/mobile browser journeys pass on one reviewed head. A subsystem receipt from another head is not release evidence.

## Separate products

Multiplayer and real-time netcode, cloud identity and sync, marketplace economics, publisher signing, cinematic media, orchestral score, high-resolution painted illustration, and hosted discovery remain separate products. They are not hidden prerequisites for the local-first cartridge runtime.
